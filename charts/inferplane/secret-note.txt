This chart does NOT create any Kubernetes Secret. By design (§7), all sensitive
material is referenced, never generated by the chart.

Before installing, create the Secret named by .Values.secrets.existingSecret
(default: "inferplane-secrets"). It is consumed by the Deployment via
`envFrom.secretRef` and must contain at least:

  ANTHROPIC_API_KEY        upstream Anthropic key (for the anthropic-direct provider)
  INFERPLANE_ADMIN_TOKEN   admin-plane bearer token (for /admin/keys)

Example (do not commit real values):

  kubectl create secret generic inferplane-secrets \
    --from-literal=ANTHROPIC_API_KEY=sk-ant-... \
    --from-literal=INFERPLANE_ADMIN_TOKEN=admin-secret

For Amazon Bedrock, prefer IRSA over a static key: set
.Values.serviceAccount.annotations."eks.amazonaws.com/role-arn" and configure a
bedrock provider with auth.mode=irsa in .Values.config (no key in the Secret).

This file has a .txt extension so Helm does not render it as a manifest.
