# internal Module

## Role
Private gateway internals — everything that is not a provider or a public package.
Packages are leaf-oriented to avoid import cycles (`principal`, `metrics`, `governance`
are leaves that others depend on).

## Key Packages
- `server/` — HTTP data plane + admin plane; `usageapi/` `GET /v1/usage` self-service handler (ADR-021 — data-plane, behind `KeyAuth`; `Governor.UsageOf` reports the caller's per-key + team budget/quota, integer µUSD, unlimited dims null, never echoes `key_id`; nil governor → ungoverned payload, never a panic); `anthropicapi/`, `openaiapi/` ingress handlers (`SetTeamPolicy`, D6/D7 ADR-019/ADR-020 — a fresh-per-request team-record lookup, same posture as `Governor.SetTeamLookup` but for non-governance per-team overrides: today's guardrail override, D7's region-lock), `adminapi/` ingress handlers; `adminui/` embedded key console (`/admin/ui/`, data-free static assets, ADR-001); `configapi/` read-only secret-free topology view (`GET /admin/config`, ADR-005), UI-write (ADR-008), plus `probe.go` `POST /admin/providers/test` (full-admin-only draft connection probe — resolves the ref server-side, SSRF-guarded `DialContext`, stateless) + `catalog.go` `GET /admin/providers/catalog` (typeahead) (ADR-014); `auditapi/` `GET /admin/audit/verify` per-sink chain check (ADR-003 #2); `adminapi/whoami.go` `GET /admin/whoami` secret-free resolved identity (dedicated DTO, PII-free) for self-service key issuance (ADR-010); `adminapi/teams.go` `GET /admin/teams` (any AdminAuth identity) + `PUT`/`DELETE /admin/teams/{name}` (full-admin only, ADR-016 D3) team governance records, `GET /admin/users` derived read-only owner projection; `adminapi/alerts.go` `GET /admin/alerts/recent` (full-admin only, D5b/ADR-017) recent budget-alert webhook fires; `analyticsapi/logs.go` `GET /admin/logs` (full-admin only, D4/ADR-018) recent request events (id-keyset paginated) + `adminapi/bodies.go` `GET`/`DELETE /admin/bodies/{ref}` (full-admin only) fetch/erase a captured body — GET emits `body_accessed` (deduped), DELETE emits `body_deleted`, both carry `record_ref` never `body_ref`, `ErrGone`→410 tombstone; `auth.go`, `adminauth.go`, `tls.go`, `metricsapi.go`.
- `router/` — model→provider resolution (reads topology from `live.Holder`, one snapshot per `ResolveChain`), priority fallback, per-provider circuit breaker keyed by identity (`breaker.go`, pruned on reload). `Canonical(model)` (ADR-021) loads one snapshot and delegates to `live.State.Canonical` — ingress handlers call it BEFORE the allow-list check so RBAC/audit/metrics/pricing all key off the canonical name. `Allows(p, model)` (ADR-021) is the RBAC check to use instead of bare `keystore.Principal.Allows` in any handler that also canonicalizes: it canonicalizes `p.AllowedModels` entries too, so an allow-list holding only an alias grants its canonical target rather than a permanent lockout (still an exact match — no broadening to unrelated models). `ChainTarget.Region` (D7, ADR-020) carries the target provider's region label (`live.State.Region`, any provider type); `FilterRegions(chain, allowed)` is a pure function — the router itself has no `Principal` in scope, so the ingress handler calls it right after `ResolveChain` with the calling team's `TeamRecord.AllowedRegions`.
- `governance/` — `Governor` (PreCheck/Settle); `UsageOf(team, keyID, kp)` (ADR-021) is the read-only companion — combines `policyOf` + `bud.Spent` + `lim.QuotaUsed` into a `UsageStatus` (per-key + team budget/quota, unlimited dims nil), no debit, no state change; `fromconfig.go` maps config → policy (USD→µUSD) via the shared `PolicyFromLimits` helper. `SetTeamLookup` installs an optional dynamic team-policy source (D3, ADR-016) consulted BEFORE the static config map on every call — a DB record wins over a config entry of the same name; nil lookup (default) is unchanged config-only behavior. `SetBudgetNotify` installs an optional per-team budget-alert hook (D5b, ADR-017), called from `Settle` after every TEAM budget debit (never per-key — `key_id` can't be a label) with the post-debit spend and configured limit.
- `keystore/` — virtual-key `Store` (SQLite), `Principal`, RBAC `Allows()`; `TeamStore` (separate interface, D3/ADR-016) — `teams` table CRUD (`UpsertTeam`/`GetTeam`/`ListTeams`/`DeleteTeam`), consumed by the assembly's `Governor.SetTeamLookup` wiring and `adminapi/teams.go`. `TeamRecord.GuardrailID`/`GuardrailVersion` (D6, ADR-019) — per-team Bedrock Guardrail override; the first ALTER-TABLE migration `teams` has ever needed (it shipped as a brand-new table under D3). `TeamRecord.AllowedRegions` (D7, ADR-020) — per-team region lock, one more column on the same migration list; fail-closed against an unlabeled target.
- `audit/` — single-writer hash-chain writer, WAL, `verify.go`, metrics hooks; `HeadHash()` race-safe chain-head snapshot + `Anchorer` interface; `audit/s3anchor/` opt-in S3 Object Lock anchoring (tamper-evident → tamper-resistant, ADR-012). `Record.BodyRef`/`RecordRef` (D4/ADR-018) appended at struct END (omitempty pointers — mixed-version chains verify byte-identically); the chain stays content-free, `BodyRef` is only an opaque pointer.
- `bodystore/` — opt-in request/response body capture (D4, ADR-018): bodies live OUTSIDE the audit chain in a separate mutable/deletable/TTL+size-capped/encrypted store. `Recorder.Capture` (copy-only, off the request path; queue-bounded, drop-never-block worker) mints a ULID `body_ref`; envelope AEAD (per-record data key wrapped by a config-ref master key, `crypto.go`); `Fetch` is fail-closed (any decrypt/miss → `ErrGone`→410 tombstone, never plaintext). Two backends (`sqlite.go` default / `postgres.go` HA) — no lease/fencing needed (per-replica ULID writes + idempotent `Purge`). Streaming responses captured request-only.
- `pricing/` — integer microUSD table, round-half-even (`math/big`), bundled defaults.
- `limiter/`, `budget/` — in-memory two-phase governance stores with injectable clocks. `budget.BudgetStore.Spent(key, window)` (D5b, ADR-017) reads current-window spend for the budget-utilization gauge and alert evaluation — mirrors `limiter.LimiterStore.QuotaUsed`.
- `alert/` — budget-alert webhook emitter (D5b, ADR-017): `Notifier.Observe(team, spentMicros, limitMicros)` evaluates a team's utilization ratio against configured thresholds and fires a fire-and-forget JSON webhook POST on each newly-crossed threshold (dedup per team; re-arms on a ratio drop, i.e. a window rollover). Keeps an in-memory ring of recent fires (`Recent()`) for `GET /admin/alerts/recent`. Leaf package (stdlib only); per-instance state (ADR-013 caveat).
- `metrics/` — Prometheus registry + GenAI collectors + nil-safe hooks.
- `openai/` — OpenAI ⇄ canonical conversion.
- `adminauth/` — admin-plane identity leaf (ADR-004): shared `IsOIDCBearerShape` predicate (config guard == middleware routing), groups→team `Resolve`, go-oidc ID-token `Verifier` (lazy discovery, negative cache, alg pin, aud/azp, ±60s skew).
- `live/` — reloadable topology generation (providers + routes + pricing) as one immutable `State` behind an atomic `Holder` (ADR-006); `BuildState` is the topology-only builder. `State.Canonical(name)` (ADR-021) resolves a config-declared model alias to its canonical name (identity for non-aliases); the alias→canonical map is built in `NewState` (so it flows through `BuildState`) and `Route` still accepts canonical names only. (imports only config/providers/pricing — import-guard tested). Hot-reload swaps the generation; governance counters/keystore/audit/breaker persist.
- `providerstore/` — opt-in DB-authoritative provider/model topology store (ADR-008, Stage 2 of ADR-005). SQLite, Postgres-portable TEXT-only DDL: `providers` (refs only — **no secret column**; `guardrail_id`/`guardrail_version` TEXT columns, D6/ADR-019 — a DB-registered Bedrock provider's own default Guardrail, mirroring the file-config and per-team paths), `model_targets` (ordered routes), `meta` (durable `seeded` marker — seed-once is marker-gated, never row-count, so deleting all providers never resurrects). `Overlay`/`OverlayFrom` build the effective config (file + DB topology, refs unresolved) — both directions thread the guardrail fields, so a config-file-declared bedrock provider's guardrail survives `SeedIfEmpty`'s file→DB import; `configapi/write.go`'s `ParseProviderWrite` validates guardrail shape/format/type-is-bedrock (same checks the team and file-config paths each enforce). UI writes go build-once-swap-once through the assembly's `reloadMu` (see `cmd/inferplane` gateway `writeMutation`); secrets never enter the store (refs only).
- `config/` — config loading + secret-ref resolution (inline secrets rejected); OIDC block validation (https issuer, mandatory client_id, JWT-shaped static tokens rejected).
- `principal/` — request-scoped principal context (leaf, breaks import cycles).
- `tracing/` — opt-in OpenTelemetry seam (ADR-011): owns the OTel SDK import (Init builds the OTLP exporter + TracerProvider + W3C propagator) and exposes Start/Extract/Inject/TraceID/SetGenAI* helpers. No-op default (library no-op tracer + `enabled` guard). Own `Config` mirror (imports no config). Handlers start one span per request, set GenAI-semconv attributes, inject traceparent upstream, and fill audit `trace_id`.
- `filter/` — request-transform filter seam (ADR-009): `RequestFilter` interface + name registry + `Masking` (resolved per-team decision). Core imports this; concrete filters live under `plugins/<name>/` and register via blank import (like providers). Leaf (imports only `sort`).

## Rules
- Pre-check BEFORE billing, settle AFTER. `on_exceeded` block wins on tie.
- Cost is integer microUSD — never float. Use `math/big` round-half-even.
- `count_tokens` must never return non-200.
- Keep `principal`, `metrics`, `governance` import-cycle-free (they are depended on widely).
- Metric labels are config-bounded; never label with raw client input (use the `_rejected` sentinel on pre-resolution rejects).
